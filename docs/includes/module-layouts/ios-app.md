```shell
my-ios-app/
├─ src/
│  ├─ KotlinCode.kt # (1)!
│  ├─ EntryPoint.swift # (2)!
│  ╰─ Info.plist
├─ module.yaml
╰─ module.xcodeproj # (3)!
```

1. Kotlin code is optional. Everything could come from dependencies.
2. The entrypoint of iOS apps must be a `@main` struct in a Swift file.
   [Read more](/user-guide/builtin-tech/ios.md/#application-entry-point)
3. An Xcode project is required but it can be automatically generated by Amper the first time.
   [Read more](/user-guide/builtin-tech/ios.md#xcode-project)
